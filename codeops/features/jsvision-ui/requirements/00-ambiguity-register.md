> **Scope**: The `@jsvision/ui` feature-set (reimagined Turbo Vision UI layer). This
> register grows per RD; entries below cover **RD-01 (Reactive core)**,
> **RD-02 (Layout engine)**, and **RD-03 (View/Group spine)**.

Every decision in an RD back-references its `AR #` here. AR-01‚Ä¶AR-04 are explicit
user choices (offered as multiple options, user selected); AR-05‚Ä¶AR-11 are
single-dominant-option decisions (no second viable alternative ‚Äî recorded for
traceability); AR-12 is a housekeeping decision; AR-13‚Ä¶AR-18 resolve the under-specified
edges surfaced by the RD-01 preflight (PF-001‚Ä¶PF-009) ‚Äî all accepted by the user 2026-06-29.
AR-19‚Ä¶AR-29 cover **RD-02 (Layout engine)**: AR-19 inherits the architecture settled in
**ADR-008** (dominant); AR-20‚Ä¶AR-29 are explicit user choices made during the RD-02
`add_requirement` interview (2026-06-29), the subset boundary ADR-008 deferred to requirements.
AR-30‚Ä¶AR-42 cover **RD-03 (View/Group spine + DrawContext + theming)**, made during the
RD-03 `add_requirement` interview (2026-06-29): AR-30‚Ä¶AR-36 are explicit user choices
(multiple viable options, user selected the recommended one); AR-37‚Ä¶AR-42 are
single-dominant-option decisions recorded for traceability (geometry reuse, composition
model, DrawContext surface, concrete surface, z-order, render root). AR-43‚Ä¶AR-46 resolve the
under-specified edges surfaced by the **RD-03 preflight** (PF-001/PF-002/PF-003/PF-005,
`00-preflight-report.md`) ‚Äî all accepted by the user 2026-06-29.
