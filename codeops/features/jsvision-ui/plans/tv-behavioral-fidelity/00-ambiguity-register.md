| PA-10 | Status · release target (runtime) | On mouse-up, which item's command emits — the one pressed, or the one under the release point? | (a) **the item under the RELEASE point** — TV `handleEvent` (`tstatusl.cpp`) tracks the highlight to the item under the cursor during the drag and on up emits that item's command if enabled (release off all items ⇒ no emit); regardless of which item the press began on; (b) press-anchored "same item as pressed" (rejected by the user — AR-88's paraphrase was stricter than TV). **Corrects AR-88's "same item" wording to the faithful `tstatusl.cpp` behavior.** | **(a) item under the release point (TV-exact)** | ✅ Resolved (user, runtime — corrects AR-88) |
| PA-11 | Left-grow · left-edge lower clamp (runtime) | PA-7's formula bounded the left edge with `anchorRight−maxW+1`, but the WM has no per-window **max-width** concept (the SE `applyResize` floors at `MIN_WIDTH` and is otherwise unbounded — overflow allowed, RD-02 AR-28).
