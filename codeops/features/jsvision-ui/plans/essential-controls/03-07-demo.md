A headless, dispatch-driven ASCII walkthrough (PA-13/AR-98) — the RD-06 acceptance vehicle, mirroring
`demo:view`/`demo:events`/`demo:shell`. New: `packages/examples/controls-demo/main.ts` (run via
`demo:controls`) + `packages/examples/test/controls-demo.e2e.test.ts`.

## Composition (PA-13)
A `Window` (blue) holding a `Group` form — controls self-color via their roles; no gray container (the
gray `Dialog` is RD-11):
- a `Label` `~N~ame` linked to an `Input` (`validator: filter('A-Za-z ')`, two-way `name` signal);
- a `CheckGroup` `[~B~old, ~I~talic]` (`Signal<boolean[]>`);
- a `RadioGroup` `(~L~eft, ~C~enter, ~R~ight)` (`Signal<number>`);
- a `Button` `~O~K` (`command: 'ok'`, `default: true`) + a command spy.

## Steps (an ASCII frame printed after each)
1. **Frame 1** — the form composed; the `Input` focused (`inputSelected`), the `[ OK ]` shadowed button.
2. **Frame 2** — type `Al3x` into the `Input`: the `3` is rejected live by `filter` ⇒ value `Alx`
   (validator live-reject visible).
3. **Frame 3** — `Tab` to the `CheckGroup`; `Space` toggles `Bold` ⇒ ` [X] Bold`.
4. **Frame 4** — `Tab` to the `RadioGroup`; `↓` selects `Center` ⇒ ` (•) Center`, others cleared.
5. **Frame 5** — `Tab` to the `Button` (`buttonFocused`); `Space` (or `Enter` as default) ⇒ the spy
   records `'ok'` (command emitted via `ev.emit`, PA-1).
6. Print a closing line.

## e2e assertions (`controls-demo.e2e.test.ts`)
- exit code 0, non-empty stdout; contains the `Label`/`Button` text, the `[X]` and `(•)` markers, the
  `◄`/`►` or field glyphs, and a narration line proving the `filter` rejection + the `'ok'` emit.

## Wiring
- The demo builds a real `createApplication` (or a bare loop + render root) and drives synthetic
  `dispatch()` events; the `Button`'s command reaches a `CommandSpy` post-process view via `ev.emit`.
- Run via a `demo:controls` script in `packages/examples/package.json` (tsx), like the other demos.

## Testing Requirements
- The e2e is the integration test; it asserts the narration + key glyphs (above). No new unit tests here
  (the controls' own spec/impl suites cover behavior).
