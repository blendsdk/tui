## Existing Implementation

### What Exists

The repository currently holds a working **Ink + React Turbo Vision "hello world"
prototype** published as package `turbo-hello` (v1.0.0). It already bypasses Ink's
renderer for painting (a custom cell buffer + ANSI serializer) but still uses Ink/React
for the host plumbing and component tree. Per **PL-1**, this entire prototype is
**archived** and `src/engine/` starts on a clean slate.

Key observations grounding the plan:
- **Not a git repository** (`git init` required ã PL-3). No GitHub remote.
- `package.json` name is `turbo-hello`, with runtime deps `ink` + `react` ã both must be removed (AR-2, AC-4).
- `tsconfig.json` is JSX/React-flavoured (`jsx: react-jsx`, `jsxImportSource: react`), `declaration: false` ã must be retargeted (no JSX, `declaration: true`, declaration maps).
- Existing tests are `*.impl.test.ts` (good ã already matches the impl naming convention), run via `tsx --test src/app/*.test.ts`.
- A prior archive already exists: `_archive/turbo-hello-clone-v1-2026-06-26.tar.gz` (the same prototype, compressed). PL-5 adds a **browsable** copy of the current `src/` for inspiration.

### Code Analysis

Ink/React coupling is confined to exactly three files (`src/index.tsx`,
`src/app/App.tsx`, `src/tui/useScreenSize.ts`); the rest of the prototype is pure
TS. This confirms the clean-slate move is low-risk: nothing in the *kept* toolchain
depends on Ink, and the reusable algorithms remain available (archived) for the
later RDs that own them.

Current `tsconfig.json` already uses `module/moduleResolution: NodeNext`, `strict`,
`noUnusedLocals`, `noUnusedParameters`, and `sourceMap` ã so the strict/ESM posture
is partly in place; the plan tightens it (add `declaration`, drop JSX).

## Gaps Identified

### Gap 1: Package identity & packaging fields
**Current Behavior:** `turbo-hello` app package, ink+react runtime deps, a `bin`, no `exports`/`files`/`sideEffects`, `declaration:false`.
**Required Behavior:** `@blendsdk/tui` v0.1.0 ESM library, zero runtime deps, `exports` map, `files` allowlist, `sideEffects:false`, ships `.d.ts` + maps (AR-6, AR-20, AC-3, AC-4).
**Fix Required:** Replace `package.json` and `tsconfig.json`; remove `bin`; add packaging fields.

### Gap 2: No clean foundation entry point
**Current Behavior:** No `src/engine/`; entry is a React/Ink `index.tsx`.
**Required Behavior:** `src/engine/index.ts` single public entry exporting `VERSION` (PL-1, PL-7).
**Fix Required:** Archive prototype `src/`; create `src/engine/{index.ts,version.ts}`.

### Gap 3: Toolchain ã lint + CI + dependency policy
**Current Behavior:** No linter, no CI, no dependency-policy guard.
**Required Behavior:** ESLint + Prettier (PL-2), GitHub Actions matrix (PL-3), `check-no-native-deps` guard (AC-6).
**Fix Required:** Add `eslint.config.js`, Prettier config, `.github/workflows/ci.yml`, `scripts/check-no-native-deps.mjs`.

### Gap 4: Not a git repo
**Current Behavior:** No `.git`; `/gitcm` commits and CI both require git.
**Required Behavior:** Initialised git repo (PL-3).
**Fix Required:** `git init` (+ initial commit handled via `/gitcm` per exec_plan).
