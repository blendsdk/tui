# Package & Build: RD-01 Scaffolding & Toolchain

> **Document**: 03-01-package-and-build.md
> **Parent**: [Index](00-index.md)

## Overview

Defines the package identity, ESM packaging surface, the TypeScript build, and the
clean `src/engine/` foundation entry point. This is the core of AC-1, AC-3, AC-4,
AC-5.

## Architecture

### Current Architecture
`turbo-hello` app package with `ink`+`react` runtime deps, a `bin`, JSX-flavoured
`tsconfig`, `declaration:false`, no `exports`/`files`/`sideEffects`. (See
[02-current-state.md](02-current-state.md).)

### Proposed Changes
A clean-slate ESM library: rename + repackage, retarget the compiler, archive the
prototype `src/`, and create `src/engine/{index.ts,version.ts}`.

### `tsconfig.json` (replaced)

```jsonc
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "outDir": "dist",
    "rootDir": "src",
    "strict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "forceConsistentCasingInFileNames": true,
    "skipLibCheck": true
  },
  "include": ["src"],
  "exclude": ["src/**/*.test.ts"]
}
```

Changes vs current: **remove** `jsx`/`jsxImportSource`/`esModuleInterop`; **add**
`declaration` + `declarationMap` (AR-20, AC-1, AC-3). `noUnusedLocals`/
`noUnusedParameters` retained ŌĆö they back AC-5. Test files are excluded from the
shipped build (`exclude`), so `dist/` carries no test code (AC-3).

### `src/engine/version.ts` (new)

```ts
/**
 * The published package version of `@blendsdk/tui`.
 *
 * Kept in sync with `package.json#version` (PL-6, PL-7). A test asserts the two
 * match so a release bump can never silently desync the runtime constant.
 */
export const VERSION = '0.1.0';
```

### `src/engine/index.ts` (new ŌĆö single public entry point)

```ts
/**
 * `@blendsdk/tui` ŌĆö public entry point of the SDK foundation.
 *
 * During RD-01 (scaffolding) the only export is {@link VERSION}; the renderer,
 * input, host, and capability subsystems are added by later RDs (RD-02ŌĆ”RD-08),
 * each re-exported from here so consumers import everything from `@blendsdk/tui`.
 */
export { VERSION } from './version.js';
```

> Foundation-first layering with a single public entry point (AR-2). The `.js`
> extension in the import specifier is required by NodeNext ESM resolution.

### `LICENSE` (new)
Standard MIT license text, copyright holder per the project owner (AR-6).

### Archive step (PL-1, PL-5)
Move the prototype out of the build before creating `src/engine/`:
- `_archive/prototype-2026-06-27/src/` ŌåÉ current `src/`
- Remove stale `dist/` (regenerated by `tsc`).
- The existing `_archive/turbo-hello-clone-v1-2026-06-26.tar.gz` is left untouched.

## Integration Points
- `exports`/`types` point at `dist/engine/index.*` ŌĆö the build output of the entry point.
- Every later RD adds modules under `src/engine/**` and re-exports public symbols from `src/engine/index.ts`.

## Error Handling

| Error Case | Handling Strategy | Ref |
| ---------- | ----------------- | --- |
| Consumer uses `require('@blendsdk/tui')` | No `require` condition in `exports` ŌåÆ Node throws `ERR_REQUIRE_ESM` (clear ESM error) | AC-1, AR-6 |
| `VERSION` desyncs from `package.json` | Spec test asserts equality; release bump must update both | PL-7 |
| Unused import/var/param introduced | `tsc` fails under `noUnused*` | AC-5 |

> **Traceability:** see [`00-ambiguity-register.md`](00-ambiguity-register.md).

## Testing Requirements
- Spec: `VERSION` is exported and equals `package.json#version` (ST-1, ST-2).
- Spec: `npm pack --dry-run --json` file set = `dist/**` + `package.json` + `README.md` + `LICENSE`; excludes `src/`, tests, `node_modules` (ST-3, ST-4).
- Spec: `package.json` declares `type:module`, `sideEffects:false`, `engines.node>=18`, an `exports` map, empty `dependencies` (ST-5).
- Spec/e2e: packed tarball installs and `import { VERSION }` resolves with `.d.ts` present; `require` fails (ST-6, ST-7).
- Spec/boundary: a file with an unused import fails `typecheck` (ST-8).
